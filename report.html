PROJECT DOCUMENTATION
STICK BRAWL
Fuzzy Logic Combat AI System  ·  Full Technical Report
1. Introduction

STICK BRAWL is a fast-paced, competitive 2D arena fighter that seamlessly blends precise melee combat with arcane spellcasting. Designed for both intense multiplayer clashes and grueling solo gauntlets, the game tracks combat performance and aggregates the data into this comprehensive analytical report.

At the core of this project is a Fuzzy Logic Inference System that powers the opponent AI — enabling it to make intelligent, human-like combat decisions in real time based on the current game state.

2. Objective

The primary objective varies by gamemode:

  • Versus Mode (1P vs AI / 2P): Deplete your opponent's health bar to zero before they do the same to you.
  • Story Mode: Survive an endless gauntlet of increasingly difficult enemy waves for as long as possible.
3. Gameplay System

Players must balance aggression and defense utilizing four core mechanics:

  • Melee Combat: Chain together rapid light and heavy physical strikes to overwhelm the opponent and build your multiplier.
  • Arcane Spells: Cast powerful ranged abilities. Spells consume mana and have strategic cooldowns that must be managed.
  • Combo System: Landing consecutive hits without taking damage builds a combo multiplier, significantly increasing damage output. The combo drops immediately upon taking a hit.
  • Parries: Precisely timed blocks negate incoming damage and briefly stun the attacker, creating a critical opening for a counter-attack.
4. System Rules & Conditions
  • A standard match concludes immediately when a combatant's HP reaches zero. The battle engine instantly logs the victor and combat stats to the database.
  • Damage output, hits landed, spells cast, and parries are continuously tracked and aggregated to provide lifetime performance metrics.
  • In Story Mode, progressing to a new wave restores a small portion of the player's health. The match ends only when the player is defeated. The highest wave reached is permanently recorded on the analysis board.
5. Fuzzy Logic AI — System Overview

The opponent AI is implemented using a Mamdani Fuzzy Inference System (FIS). Unlike traditional rule-based AI that uses hard thresholds (e.g. "if HP < 30 then run"), fuzzy logic allows the AI to reason with degrees of truth — producing smoother, more realistic behaviour that closely resembles human decision-making.

The system is classified as Multiple-Input Single-Output (MISO):

  • 4 crisp inputs are read from the live game state every tick.
  • Each input is fuzzified into linguistic membership degrees.
  • A rule base of 10 IF-THEN rules evaluates the fuzzified values.
  • The fired rules are aggregated and defuzzified into 1 crisp output (0–100).
  • The output is mapped to one of three combat actions: DEFEND, HARASS, or MELEE.
6. Input Variables & Membership Functions

The AI reads four input variables from the game state each frame:

  • Distance (0–700 px) — pixel distance between the AI and the player.
    Linguistic terms: CLOSE · MEDIUM · FAR
    CLOSE = trapMF(0, 0, 80, 200)  |  MEDIUM = triMF(80, 200, 350)  |  FAR = trapMF(280, 400, 700, 700)
  • Health (0.0–1.0) — AI current HP as a ratio of max HP.
    Linguistic terms: LOW · MEDIUM · HIGH
    LOW = trapMF(0, 0, 0.20, 0.35)  |  MEDIUM = triMF(0.25, 0.50, 0.72)  |  HIGH = trapMF(0.60, 0.75, 1.0, 1.0)
  • Mana (0.0–1.0) — AI current mana as a ratio of max mana.
    Linguistic terms: LOW · MEDIUM · HIGH
    Same membership function shapes as Health.
  • Threat (0.0–1.0) — danger level from incoming enemy projectiles (proximity-weighted).
    Linguistic terms: NONE · MODERATE · HIGH
    NONE = trapMF(0, 0, 0.20, 0.40)  |  MODERATE = triMF(0.30, 0.50, 0.70)  |  HIGH = trapMF(0.55, 0.75, 1.0, 1.0)

Two membership function shapes are used: triMF (triangular — peaks at one point) and trapMF (trapezoidal — flat top between two points). Both return a value in [0, 1] representing degree of membership.

7. Fuzzy Rule Base (10 Rules)

The knowledge base contains 10 IF-THEN rules encoding expert combat strategy. The AND operator uses the MIN method (Mamdani intersection). Each rule fires with a strength proportional to how well its antecedents are satisfied.

MELEE rules (aggressive pursuit):

  • R1: IF dist IS CLOSE AND hp IS HIGH → MELEE RUSH (out: 100)
  • R2: IF dist IS CLOSE AND hp IS MEDIUM → MELEE (out: 85)
  • R3: IF mana IS LOW AND hp IS HIGH → MELEE (conserve mana) (out: 80)
  • R4: IF dist IS MEDIUM AND hp IS HIGH AND mana IS LOW → MELEE (out: 75)

HARASS rules (ranged pressure):

  • R5: IF dist IS FAR AND mana IS HIGH → RANGED HARASS (out: 50)
  • R6: IF dist IS MEDIUM AND mana IS HIGH AND hp IS HIGH → HARASS (out: 60)
  • R7: IF threat IS MODERATE AND hp IS MEDIUM → CAUTIOUS HARASS (out: 45)

DEFEND rules (evasion & survival):

  • R8: IF threat IS HIGH → DEFEND (out: 5)
  • R9: IF hp IS LOW → DEFEND (out: 10)
  • R10: IF hp IS LOW AND threat IS MODERATE → FULL DEFEND (out: 0)
8. Inference & Defuzzification

Fuzzification — Each crisp input is passed through its membership functions to produce a set of linguistic degrees. For example, a distance of 150px produces: CLOSE = 0.33, MEDIUM = 0.85, FAR = 0.0.

Rule Evaluation — Each rule's firing strength (w) is computed using MIN for AND. Only rules with w > 0.01 are considered "fired" and contribute to the output.

Aggregation — All fired rules are combined into a single set of (weight, singleton output) pairs.

Defuzzification — Centre of Gravity (COG)

COG = Σ(wᵢ × outᵢ) / Σwᵢ

The result is a crisp value in [0, 100] that is mapped to a final action:

  • 0 – 33 → DEFEND — retreat, shield, place ice wall, heal if possible.
  • 34 – 66 → HARASS — maintain distance, cast fireballs, use burst spell.
  • 67 – 100 → MELEE — close gap, punch, trigger ultimate if charged.

If no rules fire (den < 0.001), the system defaults to HARASS (COG = 50) as a neutral fallback.

9. AI Difficulty & Reaction Speed

The fuzzy inference cycle does not run every frame — it runs every N frames, where N is controlled by the difficulty setting. This simulates human reaction delay:

  • Easy: N = 14  (~4 decisions/second) — AI reacts slowly, giving the player time to act.
  • Medium: N = 7  (~9 decisions/second) — balanced, normal AI behaviour.
  • Hard: N = 4  (~15 decisions/second) — near-instantaneous reactions.

The fuzzy logic output remains the same regardless of difficulty — only the frequency of decision-making changes, not the intelligence of the decisions.

10. Knowledge Acquisition

The fuzzy rules and membership function parameters in this system were derived through a structured knowledge acquisition process involving three main sources:

  • Expert Game Design Analysis: Combat AI behaviour from established 2D fighting games was studied to identify common strategic patterns — when to attack, retreat, or use special abilities based on health, distance, and resource levels.
  • Domain Knowledge Encoding: Linguistic variables and thresholds were defined based on gameplay mechanics. For example, "CLOSE" distance (0–200px) is the effective melee range; "LOW" health (<35%) reflects a critically vulnerable state requiring defensive action.
  • Iterative Playtesting: Rules were refined through repeated test scenarios, observing AI decisions under various game states and adjusting membership function boundaries and rule weights until the AI exhibited natural, balanced behaviour across all three archetypes (Mage, Brawler, Ninja).
11. Market Analysis

Target Users

  • Students and academics studying Artificial Intelligence, Fuzzy Logic, or Game AI — as an interactive educational demonstration.
  • Indie game developers seeking to implement non-scripted, adaptive AI opponents in 2D action games.
  • Casual gamers who enjoy competitive fighting games with intelligent AI adversaries.

Problem Solved Using Fuzzy Logic

Traditional game AI relies on hard-coded if-else thresholds (e.g. "if HP < 30, run away") which produce predictable, robotic behaviour. This system replaces that with fuzzy logic, enabling the AI to make graduated, context-aware decisions — behaving more like a human player who weighs multiple factors simultaneously.

Value & Benefit

  • Demonstrates real-time Mamdani FIS in a visually transparent, interactive environment.
  • Provides live membership function graphs, system model diagrams, and a debug panel — making the fuzzy reasoning process fully observable.
  • Serves as a teaching tool: instructors can use the game to explain fuzzification, rule evaluation, and defuzzification with concrete visual feedback.

SWOT Analysis

  • Strength: Fully functional, visually rich, transparent AI system with real-time debugging tools. Complete game loop with multiple modes and characters.
  • Strength: All fuzzy logic components (MF graphs, system diagram, debug panel) are built into the game itself — no external tools needed.
  • Weakness: Currently MISO (single output axis). A MIMO system with multiple independent outputs would allow richer AI behaviour.
  • Weakness: Rules are manually crafted — a machine learning approach (neuro-fuzzy) could auto-optimise rule weights.
  • Opportunity: Expandable to include more input variables (combo state, position, cooldowns) and more rules for deeper strategic depth.
  • Opportunity: Can be packaged as an educational module for university AI courses.
  • Threat: Neural network-based game AI (e.g. reinforcement learning) is becoming more common and may outperform hand-crafted fuzzy systems in complex scenarios.

Growth Roadmap

  • Phase 1 (Current): Working prototype with 3 characters, 3 maps, 10 fuzzy rules, live AI transparency panel.
  • Phase 2: Expand rule base (20+ rules), add combo-state and position inputs, implement MIMO output (aggression + movement + spell priority as separate axes).
  • Phase 3: Neuro-fuzzy hybrid — use reinforcement learning to auto-tune membership function parameters from match data.
  • Phase 4: Package as a browser-hosted educational tool with guided tutorials on fuzzy logic concepts, deployable in university AI labs.
12. Conclusion

This project successfully demonstrates the application of a Mamdani Fuzzy Inference System in a real-time, interactive game environment. The AI opponent in STICK BRAWL uses four crisp inputs — distance, health, mana, and threat — fuzzified through triangular and trapezoidal membership functions, evaluated against a rule base of ten IF-THEN rules, and defuzzified using the Centre of Gravity method to produce a single combat action.

The system exhibits several key strengths:

  • The AI behaviour is smooth and graduated, avoiding the robotic predictability of threshold-based logic. At 60% health, the AI does not instantly switch modes — it gradually shifts weight between HARASS and DEFEND based on all four inputs simultaneously.
  • The reasoning process is fully transparent — players can observe the live fuzzy panel, membership function graphs, and system model diagram at any time during gameplay by pressing P.
  • The system meets all Progress 2 requirements: Knowledge Base, Knowledge Acquisition, Fuzzification, Inference Engine, Aggregation, and Defuzzification are all implemented and integrated.

The main limitation is the MISO architecture — all combat decisions collapse into a single output axis. Future work would explore MIMO design, neuro-fuzzy parameter learning, and a broader rule base incorporating additional game-state variables.

Overall, this project confirms that fuzzy logic is a highly appropriate and effective approach for game AI decision-making — offering a strong balance of interpretability, flexibility, and real-time performance.
